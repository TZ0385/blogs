# 钓鱼邮件攻击

社会工程学是渗透测试中的重要环节，钓鱼邮件是其中最常用的攻击手段之一。本文档涵盖钓鱼邮件的设计、制作和投递技术。

## 3 钓鱼策略设计

### 3.1 常见钓鱼场景

| 场景类型 | 描述 | 目标角色 |
|----------|------|----------|
| 密码到期提醒 | 伪装IT部门要求更新密码 | 全员 |
| 邮箱配额不足 | 声称邮箱空间不足需清理 | 全员 |
| 发票/付款通知 | 伪装财务发送发票 | 财务人员 |
| 会议邀请 | 伪装HR或领导发送会议邀请 | 管理层 |
| 快递通知 | 伪装快递公司发送取件通知 | 行政/收货人 |
| 账户异常 | 声称账户存在异常登录 | 特定用户 |
| 奖品中奖 | 虚假中奖信息 | 广撒网 |
| 供应商通知 | 伪装供应商更新银行账户 | 财务 |

### 3.2 邮件主题设计原则

```
优秀钓鱼邮件主题特征:
├── 紧迫性 - "24小时内必须处理"
├── 权威性 - 伪装领导/IT/HR
├── 相关性 - 与目标工作相关
├── 好奇心 - 引起点击欲望
└── 简洁性 - 不引起怀疑
```

#### 常用钓鱼主题模板

```text
# IT相关
[IT通知] 您的邮箱密码将于今天过期，请立即更新
[系统升级] 请在24小时内验证您的邮箱账户
[安全警告] 检测到您的账户存在异常登录

# 财务相关
【紧急】供应商银行账户变更通知
[发票] 2024年XX月发票 #XXXXX 待确认
[付款通知] 客户付款凭证-请查收

# 行政相关
[会议邀请] 关于XXX项目的紧急会议
【考勤】您的考勤异常请及时处理
[通知] 端午节放假安排及值班表

# 通用型
【已投简历】来自XXX的求职简历
[文件共享] "XXX.pdf" 已与您共享
【包裹】您的快递单号XXXXX正在派送
```

## 5 钓鱼规避检测技术

### 5.1 邮件安全机制规避

```bash
# 1. SPF/DKIM/DMARC检查
# 确保发件域名通过SPF检查
# 使用与钓鱼域名匹配的邮件服务器

# 2. 链接混淆
# URL短链接
# 恶意URL插入正常URL中间
http://target.com/go/redirect?url=http://evil.com
# HTML混淆
<a href="http://evil.com" onclick="location.href='http://target.com'">点击这里</a>

# 3. 内容规避
# 避免敏感关键词（发票、密码、账号）
# 使用图片代替文字
# Base64编码内容
```

### 5.2 反钓鱼邮件过滤规避

```text
规避关键词:
❌ 密码    -> 访问密钥/登录凭证
❌ 账号    -> 用户ID/登录账号
❌ 点击这里 -> [这里]
❌ 中奖    -> 获奖通知
❌ 发票    -> 收据/凭证

内容技巧:
1. 短邮件正文，减少被检测文本
2. 使用真实企业logo和格式
3. 添加正常邮件的签名
4. 使用与目标企业相似的域名
```

## 8 防御措施

### 8.1 邮件安全配置

```text
技术防护:
1. SPF记录配置
   v=spf1 include:_spf.target.com ~all

2. DKIM签名配置
   添加DKIM签名记录

3. DMARC策略配置
   v=DMARC1; p=quarantine; rua=mailto:dmarc@target.com

4. 邮件网关部署
   - 垃圾邮件过滤器
   - 恶意附件检测
   - URL点击前检测

5. 双因素认证(2FA)
   - 强制对所有邮箱使用2FA
   - 敏感操作二次验证
```

### 8.2 用户安全培训

```text
钓鱼识别培训:
1. 检查发件人地址
   - 是否为企业官方域名
   - 是否有细微拼写差异

2. 检查链接
   - 鼠标悬停查看真实URL
   - 警惕短链接

3. 检查内容
   - 是否要求提供敏感信息
   - 是否制造紧迫感

4. 附件谨慎
   - 不打开未知来源附件
   - Office宏默认禁用

5. 举报机制
   - 建立钓鱼邮件举报渠道
   - 定期演练测试
```
